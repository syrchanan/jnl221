#first

By default, content division elements (divs) have no padding, margin or border. So they extend horizontally as wide as possible but tightly wrap vertically around whatever's inside, like a glove.

But: Heading and paragraph elements have top and bottom margins by default to create vertical distance. They will repel HTML elements above and below until we make changes to the div itself using CSS, like adding some border or padding to the top or bottom.

Try adding a border or top/bottom padding to this div and see what happens.

#third

We're primarily concerned right now with vertical layouts — just getting our content to stack nicely on desktop and mobile screens.

Setting a max-width is helpful for this. It helps keep text, images and data visualizations from taking up the entire width of a large desktop/laptop screen. You provide a maximum width that the element, like a div, can't exceed. But it can get narrower!

In the CSS file, give this one a max-width of 700px.

#fourth

How can we get this div get horizontally centered?

Use the same max-width attribute as the div above, but set the left and right margins to "auto", which means they're calculated automatically based on the width of the browser window.

This allows the div to hang out in the middle. As the screen gets narrower, the margins will eventually shrink away to nothing and it'll take up the full width.

(This is similar to how your recipe text was centered on the webpage in an earlier in-class activity, if you go back to peek at the CSS).

#fifth

We can also have the div hang out on the far right. Use the same properties as the one above, except set the margin-left to "auto" and the margin-right to 0.
